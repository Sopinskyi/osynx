Privacy Policy

Last updated: 21 Oct 2025

GSA-X is a Google Sheets™ add-on developed by Ochyro Inc (“we,” “us,” “our”). This policy explains what data the add-on accesses and how we use it.

What the add-on does

GSA-X lets you look up Samsara assets in Google Sheets, fetch current locations, and insert deep links that open directly in Samsara.

Data we access

  • Google Sheets data you select in the spreadsheet where you run the add-on (read/write to the active sheet to populate and refresh rows).
  • Basic Google account info (userinfo.email and, if enabled, userinfo.profile) to show which account is signed in and for support diagnostics.
  • Locale (for formatting labels/timestamps).
  • External requests to the Samsara API you configure (we call https://api.samsara.com/ using your API token/Org ID).

How we use data

  • Populate and refresh the current spreadsheet’s rows with Samsara asset information at your request.
  • Generate “Open in Samsara” links for convenience.
  • Display your signed-in account in the add-on UI and format content per your locale.

Where data is stored

  • We do not operate any backend servers for GSA-X and do not store Google user data server-side.
  • Your Samsara API token and Org ID are stored locally in the document properties of the Google Sheet where you paste them. You can remove them at any time (see “Your choices & deletion”).

Sharing & selling

  • We do not sell or rent your data.
  • We only send requests to the Samsara API you configure, using your token, to fulfill your explicit actions (lookups/refreshes).

Security

All communication with Google and Samsara APIs occurs over HTTPS/TLS. Because GSA-X does not use our own servers, there is no server-side database to breach. You control the token stored in your spreadsheet.

Your choices & deletion

  • Remove credentials: In the same spreadsheet, delete the add-on’s saved token/Org ID from the settings panel (or clear the document properties).
  • Uninstall the add-on: Google Sheets → Extensions → Manage add-ons → GSA-X → Uninstall.
  • Revoke OAuth access: Google Account → Security → Third-party access → GSA-X → Remove access.

Google API Services User Data Policy (Limited Use)

GSA-X’s use of Google user data complies with the Google API Services User Data Policy, including the Limited Use requirements. We only use the minimum necessary scopes to provide the add-on features, do not transfer Google user data except as needed to fulfill the user’s request, and do not use the data for advertising.

Children

GSA-X is not intended for children under 13.
